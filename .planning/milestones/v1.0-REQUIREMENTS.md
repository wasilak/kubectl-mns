# Requirements — Milestone v1.0 Foundation & Quality

_Created: 2026-06-21_

## SAFETY — Shell Safety

- [x] **SAFETY-01**: Plugin quotes all array expansions (`"${namespaces[@]}"`, `"${actual_kubectl_args[@]}"`)
- [x] **SAFETY-02**: kubectl is invoked via bash array, not string concatenation
- [x] **SAFETY-03**: Data presence check uses `-n "$data"` instead of `! -z $data`

## BUGFIX — Bug Fixes

- [x] **BUGFIX-01**: README usage example omits redundant `kubectl` after `--`
- [x] **BUGFIX-02**: `usage()` displays `namespace-2` (typo: was `namespac-2`)
- [x] **BUGFIX-03**: `usage()` output goes to stderr instead of stdout
- [x] **BUGFIX-04**: Plugin exits with code 1 when invoked with no args (was: 0)

## SECURITY — CI Security

- [x] **SECURITY-01**: Codacy GitHub Action pinned to a commit SHA (not `@master`)

## ERRORS — Error Handling

- [x] **ERRORS-01**: Per-namespace kubectl failure is caught; script continues and reports error to stderr

## OUTPUT — Output UX

- [x] **OUTPUT-01**: Namespace label printed before each output block (`=== namespace: <ns> ===`)

## ARGS — Argument Handling

- [x] **ARGS-01**: `--context <ctx>` accepted before `--` and forwarded to every kubectl call
- [x] **ARGS-02**: `--kubeconfig <path>` accepted before `--` and forwarded to every kubectl call

## TESTS — Test Suite

- [x] **TESTS-01**: bats-core test suite exists at `test/kubectl-mns.bats`
- [x] **TESTS-02**: Test: no namespace given → defaults to `default`
- [x] **TESTS-03**: Test: multiple namespaces → one kubectl call per namespace
- [x] **TESTS-04**: Test: `--all-namespaces` stripped from forwarded args
- [x] **TESTS-05**: Test: empty kubectl args → exits 1 and prints usage
- [x] **TESTS-06**: Test: `-h` / `--help` → prints usage and exits 0
- [x] **TESTS-07**: Test: namespace failure continues to next namespace

## Requirement Outcomes

All 19 requirements shipped without adjustment. No requirements were dropped or materially changed during delivery. Two enhancements emerged during cross-AI review of Phase 2 (kubectl stderr passthrough, post-loop dangling-flag guard) and were folded into the same phase as bonus items — not tracked as separate requirements.

## Deferred Requirements (carry-forward candidates)

- Parallel namespace execution — adds output ordering complexity; sequential is intentional for now
- Release pipeline / versioning — address in v1.1
- Client-side namespace name validation (RFC 1123) — kubectl validates server-side

## Out of Scope

- `--all-namespaces` mode (stripped from args today, not added as a flag) — not requested
- Output formatting beyond namespace labels (JSON, table) — not requested
- Kubeconfig context listing / auto-discovery — out of scope for this milestone
