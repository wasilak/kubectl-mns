| T-02-02 | Tampering | `--kubeconfig <path>` value containing shell metacharacters | mitigate | Same array-exec mechanism as T-02-01. Path is treated as opaque string passed to kubectl; kubectl itself validates the file contents. |
| T-02-03 | Tampering | Namespace name containing shell metacharacters or whitespace | mitigate | Already hardened in Phase 1 â€” `namespaces+=("$item")` quotes the array assignment; expansion via `"${namespaces[@]}"` preserves element boundaries. No new surface introduced by Phase 2. |
| T-02-04 | Information disclosure | kubectl error output captured into `$data` via `2>&1` and echoed to stderr in error-continuation branch (ERRORS-01) | accept | kubectl's stderr is by design intended for the invoking user; forwarding it to the user's stderr does not cross a trust boundary.
