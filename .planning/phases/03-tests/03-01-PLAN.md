<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| test subprocess → plugin | `run "$PLUGIN"` spawns plugin as subprocess; plugin sees test's exported env vars |
| plugin → kubectl stub | Plugin resolves `kubectl` via PATH; stub receives all forwarded args |
| CI runner → bats-action | GitHub Actions fetches action code from external GitHub org |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Tampering | $STUB_DIR/kubectl stub | mitigate | stub written from repo-controlled heredoc only; mktemp dir is not world-writable; teardown() removes it |
| T-03-02 | Tampering | .github/workflows/tests.yml (unpinned actions) | mitigate | pin both `actions/checkout` and `bats-core/bats-action` to commit SHAs per SECURITY-01 policy |
| T-03-03 | Information Disclosure | STUB_CALL_LOG in /tmp | accept | log contains only kubectl arg strings (no secrets); removed in teardown(); low-value target |
| T-03-04 | Spoofing | Test assertions pass vacuously (no real behavior exercised) | mitigate | every test asserts both exit code AND content (stub log or output text); stderr tests use `2>&1` redirect so Usage text is always verified |
| T-03-SC | Tampering | bats-core/bats-action install in CI | mitigate | action pinned to SHA `5b1e60c2ee94cb1b44a616ea4b1f466f9d6e38ef` — tag immutability not relied upon |
</threat_model>
